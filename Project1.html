HTML:

Representación de un documento HTML:

Estructura de los tags:

  • DOCTYPE: Define el tipo de documento y debe ser la primera linea de un documento HTML
  • HTML : Define la raíz del documento. Contiene un HEAD y un BODY (en este orden)
    • Head: Contiene metadata del documento
      • - Title
      - Link
      - Style
      - Script (Para incluir código de JavaScript directamente en HTML)
      - Meta
      - ...
    • Body: Representa el contenido del documento

Text-level tags

  • <p>: Marca párrafos
  • <br>: Salto de linia
  • <em>: Emfatizar
  • <strong>: Marca como importante
  • <code>
  • <pre>: Permite poner el texto preformateado

Sections

  • Main tag: Representa el contenido dominante del documento (debe ser único)
  • Article tag: Contiene una composición completa en el documento

  • Section tag: Marca una sección generica del documento

  • Nav tag: Marca una sección de la página que contiene links de navegación
  • Headings: Describe los temas que preceden desde h1 a h6
  • Header tag: Representa un grupo de ayudas introductorias o de navegación
  • Footer tag: Marca la información de la sección (autores, copyright data, documentos relacionades,etc)
  • Div tag: Es una marca genérica para una sección dentro del documento

La página es un bloque de elementos el cual está hecho de elementos inline

  • Block: Representa un rectangulo horitzontal en la página
  • Inline: Representa una linea de los elementos de dentro del block.

CSS Box Model

El modelo de cajas ("Box Model") es el comportamiento CSS que hace que todos los elementos incluidos en una página web se representen mediante cajas rectangulares.

Las partes que componen cada caja son:

  1. Contenido (Content): Contenido HTML del elemento(palabras, imagenes, texto...).
  2. Relleno (Pading): Espacio libre opcional existente entre el contenido y el borde que lo encierra.
  3. Borde(Border): Línea que encierra completamente el contenido y su relleno.
  4. Imagen de fondo (background-image): Imagen que se muestra por detrás del contenido y el espacio de relleno.
  5. Margen (Margin): espacio libre entre la caja y las posibles cajas adyacentes.

FlexBox

FlexBox se basa en dos ejes: El eje principal y el eje cruzado.

Eje principal (main axis)

El eje principal está definido los siguientes tres parámetro:
  • Flex-direction: Especifica el eje principal, posee cuatro posibles valores:
    • Row o Row-rerse: El eje principal correrá a lo largo de la fila según la dirección de la línea.
    • Column o Column-reverse: El eje principal correrá desde el borde superior de la página hasta el final (según la dirección del bloque). Si no se especifica ningún flex-direction por defecto estará en columna.
  • Justify-content: Especifica la distribución de los hijos a lo largo del eje primario.
  • Flex-start: Los items flex se colocan comenzando desde el comienzo principal.
  • Felx-end: Los items se colocan comenzando desde el final principal.
  • Center:Los items flex son colocados hacia el centro de la línea. Los items flex se alinean al ras entre sí y en torno al centro de la línea
  • Space-between: Los items flex se alinean uniformemente de tal manera que el espacio entre dos items adyacentes es el mismo.
  • Align-items: establece el valor align-self sobre todos los descendientes directos de un grupo. La propiedad align-self indica la alineación de un elemento dentro del bloque que lo contiene.

Eje cruzado (cross axis)

El eje cruzado va perpendicular al eje principal. Si flex-direction del eje principal es row/row-reverse, el eje cruzado irá por las columnas y viceversa.

Element position

Un elemento si no se especifica lo contrario su posicionado es el natural, es decir estatico (static). Se pueden sacar los elementos de su flujo local y colocarlos donde queramos. Hay tres tipos de posicionado:

  • Elemento posicionado relativamente: Relative nos permite posicionar un elemento en relación a su posición estática mediante top/bottom/left/right.
  • Elemento posicionado absolutamnte: Absolute permite quitar el elemento del flujo normal. Las propiedades top, right, bottom, y left especifican el desplazamiento desde los bordes del bloque contenedor del elemento. (El bloque contenedor es el ancestro relativo al cual el elemento está posicionado).
  • Elemento posicionado fijamente: Sticky permite tratar al elemento como un posicionado relativamente hasta que su bloque contenedor cruza un límit establecido y entonces pasa a ser tratado como fijo.

The DOM

El modelo de objeto de documento (DOM) es una interfaz de programación para los documentos HTML. Facilita una representación estructurada del documento y define de qué manera los programas pueden acceder, al fin de modificar, tanto sue structura, estilo y contenido. Principalmente conecta las páginas web a scripts o lenguajes de programación.

The document object

La interfaz documento representa cualquier página web cargada en el navegador y sirve como punto de entrada al contenido de la página (el árbol DOM). La interfaz document también hereda de las interfaces Node y EventTarget.

Element queries

Se pueden buscar elementos de dentro de la página mediante diferentes métodos:

  • getElementsByTagName (tag)
  • getElementById (id)
  • getElementsByClassName (class)
  • querySelector (selector)
  • querySelectorAll (selector)

Eventos

Los eventos event son acciones u ocurrencias que suceden en el sistema que está programando y que el sistema le informa para que pueda responder de alguna manera si lo desea. Por ejemplo, si el usuario hace clic en un botón en una página web, es posible que desee responder a esa acción mostrando un cuadro de información.

Cada evento tiene un controlador (event Handler) de eventos, que es un bloque de código que se ejecutará cuando se active el evento. El oyente (event Listener) escucha si ocurre el evento y el controlados es el código que se ejecuta en respuesta a que ocurra.

EventTarget es una interfaz implementada por los objetos que pueden administrar eventos y sus "listeners". Los objetos más comunes de un evento son Element, document y window.

El event tiene siempre un elemento que se llama target (el target es el elemento que produce un evento).

  • addEventListener(event, handler): Registra un controlador de eventos de tipo event. Cuando ocurra tal evento → ejecuta tal función.
  • removeEventListener(event, handler): Elimina un detector de eventos del EventTarget.
  • dispatchEvent(event, handler): Enviar un evento a este EventTarget. Es decir, se engaña al navegador para que este crea que ha ocurrido un evento.

Forms

El elemento HTML form representa una sección de un documento que contiene controles interactivos que permiten a un usuario enviar información a un servidor web.

Como cualquier otro elemento HTML, soporta atributos globales.

  • Action: Recibe una URL que se encarga de procesar los datos del formulario.
  • Method: Método HTTP que el navegador usa para enviar el formulario (nos dice por que método vamos a enviar la información). Este método puede ser get o post:
    • post: Corresponde al método POST HTTP; los datos del formulario son incluidos en el cuerpo del formulario y son enviados al servidos.
    • get: Corresponde al método GET HTTP; los datos del formulario son adjuntados a la URL del atributo action, con un '?' como separador.

Form structure

El elemento HTML input se usa para crear controles interactivos para formularios basados en la web con el fin de recibir datos del usuario.

  • Type: type representa el tipo de control a mostrar (el tipo de entrada que queremos). Si no se especifica este atributo, su valor predeterminado es text. Los posibles valores son: button, checkbox, color, date, datetime, datetime-local, email, file, hidden, image, month, number, password, radio, range, reset, search, submit, tel, text, time, url, week.
  • Value: Determina el valor inicial de control.
  • Name: Controla la variable que se va a rellenar. Nos permite establecer al servidor que valor estamos pidiendo.

Label Tag

El elemento label es la forma formal de definir una etiqueta para un control de un formulario HTML. Se crean para estar asociados a uno de los inputs (campos).
